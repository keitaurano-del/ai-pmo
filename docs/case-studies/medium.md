# ケーススタディ：中規模企業（50〜500名）

## 仮想企業（プレースホルダ）

> Keita のレビュー後に業種・社名を確定。

- 業種: 例）中堅 SIer（受託開発、社員300名・PM/PMO 計15名）
- 従業員数: 200〜400名
- 主要事業:
- 既存の PM/PMO 体制: PMO 部門あり（5〜10名）、複数 PJ を並走

## 想定する課題

- プロジェクト横断の可視化ができていない
- PMO の報告業務が形骸化、レポート作成に追われている
- 課題・リスクの早期検知が属人的
- 複数案件で標準が揃わない

## AI PMO 導入シナリオ

### Phase 1（PoC、〜3ヶ月）
- 1〜2 PJ を選定して PoC
- レポーター + スクライブ + ウォッチャー + トリアージャー
- 既存 Jira/Backlog と連携

### Phase 2（展開、3〜9ヶ月）
- 全 PJ への段階展開（月3〜5 PJ）
- PMO 部門との役割再整理
- リスクアナリスト追加

### Phase 3（定着、9〜12ヶ月）
- ガバナンス確立、KPI 監視体制
- 卒業 or 保守契約へ

## 必要なエージェント構成

- レポーター
- スクライブ
- ウォッチャー
- トリアージャー
- リスクアナリスト

## 期待効果（KPI）

- PMO 報告業務: 60% 削減
- 課題検出リードタイム: 平均5日 → 1日以内
- PJ 状況の経営層への可視化頻度: 月次 → 週次

## 導入コスト・ROI

- 総額: ¥30M〜¥80M（12ヶ月、[[../pricing|pricing]] 参照）
- ROI: PMO 5名分の工数50%削減で1.5〜2年で回収

## リスク・前提条件

- 既存 PMO メンバーの抵抗 → チェンジマネジメント必須（[[../consultant-playbook|Phase 6]]）
- ツール多様性（PJ ごとに違う） → 連携設計に工数
