# Phase 5: 運用（PoC 3ヶ月 + 卒業判定）

## 目的

AI PMO エージェントを **日常運用** し、PoC 期間（3ヶ月）終了時に **本格展開 / 縮退 / 終了** を判定する。

## 運用カデンス

### 日次

| 時刻 | 内容 | 担当 |
|------|------|------|
| 06:00 | watcher-agent が前日 Slack/Backlog を走査、課題自動起票 | AI |
| 09:00 | Jr. PMO アナリストが新規起票課題をレビュー (15分) | 人間 |
| 18:00 | triager-agent が全オープン課題のリスコア | AI |
| 18:30 | High 課題のサマリを Slack #ai-pmo-alerts へ通知 | AI |

|------------|------|------|

## トラブル時のロールバック

「AI PMO が暴走している」「品質が許容できない」と判断したら即停止できる手順：

1. **個別エージェント停止**: cron / n8n から該当ジョブを無効化
2. **全停止**: AI PMO Slack Bot を kick、API キー失効
3. **データ保全**: 過去の生成物は GitHub 履歴に残るため失われない
4. **人手 PMO への切替**: クライアント PMO 担当が手動運用を再開

ロールバック手順書を `client-env/00-context/rollback.md` に常備。

## PoC 卒業判定（3ヶ月後）

PoC 終了 1ヶ月前から判定準備：

### 判定基準

| 観点 | 判定 | アクション |
|------|------|------------|
| 全 KPI 達成 + 高満足度 | ✅ 拡大 | Phase 2 へ（他 PJ 展開、定期契約） |
| KPI 部分達成・運用継続意思あり | 🟡 縮小継続 | スコープ縮小して継続契約 |
| KPI 未達 + 不満多い | 🔴 終了 | 縮退して期間内終了 |

### 卒業判定会議

- 参加: 経営スポンサー + PMO 室長 + Sr. コンサル + PM
- 所要: 90分
- アジェンダ:
  - KPI 実績レビュー
  - 定性フィードバック（PMO 担当・PM 全員）
  - ROI 試算
  - 次フェーズ提案（拡大 / 縮小 / 終了）
  - 契約形態切替の議論
