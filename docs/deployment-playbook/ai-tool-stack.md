# AI / ツールスタック

AI PMO サービスで使用する全ツール・SaaS の整理。提案時の見積もり根拠としても使う。

## レイヤ別構成

```
┌──────────────────────────────────────────────────┐
│  クライアント向け配信レイヤ                          │
│  Confluence / Slack / メール / Web ビューワー       │
└──────────────────────────────────────────────────┘
                       ↑
┌──────────────────────────────────────────────────┐
│  オーケストレーションレイヤ                          │
│  n8n / Zapier / GitHub Actions                  │
└──────────────────────────────────────────────────┘
                       ↑
┌──────────────────────────────────────────────────┐
│  AI エージェントレイヤ                              │
│  Claude API (Opus 4.7) + Claude Agent SDK       │
│  + 自前実装（reporter / scribe / watcher / triager）│
└──────────────────────────────────────────────────┘
                       ↑
┌──────────────────────────────────────────────────┐
│  データ統合レイヤ                                   │
│  MCP サーバー / 各 SaaS API クライアント            │
└──────────────────────────────────────────────────┘
                       ↑
┌──────────────────────────────────────────────────┐
│  データソースレイヤ                                 │
│  Backlog / Jira / Slack / Box / Zoom / TeamSpirit │
└──────────────────────────────────────────────────┘
```

## 法人向け留意点

- **データ主権**: クライアントによっては「AWS Tokyo Region のみ」「オンプレ必須」要件あり → AWS Bedrock 経由 Claude も視野
- **個人情報保護**: PII 検知層を必須化（議事録に氏名が出ても、外部送信時はマスク）
- **監査ログ**: AI への入力・出力を全件保存（GDPR / 業界規制対応）
- **モデル切替**: クライアント要望で OpenAI / Gemini への切替が起きる前提、抽象化レイヤを持つ

詳細は `docs/enterprise-considerations.md` を参照。
