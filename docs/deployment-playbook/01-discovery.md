# Phase 1: Discovery（1〜2週間）

## 目的

クライアントの **現状の PMO 業務・既存ツール・抱えている課題・対象 PJ** を把握し、
AI PMO 導入の **対象 PJ 選定** と **期待効果のベースライン** を確定する。

## 期間

- 標準: 2週間（実働 30〜40h）
- 短縮版: 1週間（実働 20h）— 緊急案件・小規模クライアント向け

## 担当

- **Sr. コンサル 1名**（インタビュー主体）
- **AI PMO エンジニア 0.5名**（ツール環境調査）
- クライアント側: PMO 室長 + 対象 PJ 候補の PM 数名

## アクティビティ

### Day 1〜2: キックオフ + 経営層ヒアリング

- 経営スポンサーとの 60分インタビュー
  - 「なぜ AI PMO を検討しているか」
  - 経営として測りたい KPI（コスト削減 / 可視化 / 標準化）
  - 想定投資額・期間
- 提出物: **Discovery キックオフ議事録**

### Day 3〜5: PMO 室現状ヒアリング

- 室長 + シニア PMO 2〜3名にロングインタビュー
- 確認項目:
  - 体制（人数・役職・経験年数）
  - 管掌範囲（同時並走 PJ 数、業種、規模）
  - **典型的な1週間の業務時間配分**（週次レポート何時間、議事録何時間 等）
  - 既存テンプレ・標準資産
  - 現状の悩み（自由形式）
  - チェンジマネジメント上のリスク（抵抗勢力、過去の失敗）
- 提出物: **PMO 室現状マップ**

### Day 4〜6: 既存ツール棚卸し

- AI PMO エンジニアが調査
- 確認項目:
  - PJ 管理ツール（Backlog/Jira/Redmine/Asana 等）と契約形態
  - ドキュメント基盤（Confluence/Notion/SharePoint）
  - コミュニケーション（Slack/Teams/Chatwork）
  - 工数管理（TeamSpirit/MFCloud/自前 Excel）
  - 経営報告フォーマット
  - **API 提供有無** と **顧客先 IT 統制**（外部 API 可否）
- 提出物: **既存ツール台帳**

### Day 6〜8: PJ ポートフォリオ棚卸し + PoC 対象選定

- 同時並走中の全 PJ をリストアップ
- 各 PJ について評価:
  - 規模（人数・期間・予算）
  - フェーズ（要件定義 / 設計 / 開発 / 運用）
  - PM の AI 受容度
  - 顧客の IT リテラシー（外部ツール導入容認度）
  - 政治的リスク（炎上案件、要人案件は避ける）
- **PoC 対象 PJ を 1 件選定**
- 提出物: **PoC PJ 選定書**（理由・代替候補・選定基準を明示）

### Day 9〜10: Discovery レポート作成 + 経営報告

- 上記すべての成果物を統合した **Discovery レポート**（20〜30 ページ）
  - エグゼクティブサマリ（3 行）
  - 現状診断
  - AI 導入余地マップ（業務別の自動化レベル予想）
  - PoC 対象 PJ 選定
  - PoC 期間・スコープ案
  - 期待効果のベースライン
  - 想定投資額
- 経営スポンサー + PMO 室長へのプレゼン（90分）
- 提出物: **Discovery 報告書（PDF）+ 経営判断**

## AI の活用ポイント

| 工程 | AI 利用 | 削減効果 |
|------|---------|----------|
| インタビュー音声の文字起こし | Whisper API or Notta | 議事録作成 80% 短縮 |
| インタビュー要約 | Claude API | 1h 録音 → 30分で要約 |
| 既存ドキュメントの構造化 | Claude API（multi-doc 読み込み） | 1日 → 1時間 |
| 業務マップ図化 | Claude + Mermaid 自動生成 | 半日 → 10分 |
| Discovery レポート初稿 | Claude API（テンプレベース） | 3日 → 半日 |

## アウトプット一覧

- Discovery キックオフ議事録
- PMO 室現状マップ
- 既存ツール台帳
- PoC PJ 選定書
- **Discovery 報告書（最終成果物）**

## 次フェーズの前提条件

✅ 経営スポンサーの正式承認
✅ PoC 対象 PJ の PM 受け入れ合意
✅ クライアント側 PoC 推進担当のアサイン
✅ 必要な API キー / ツールアクセス権の発行プロセス確認

これらが揃ったら **Phase 2: 環境セットアップ** へ。
