# AI PMO 導入支援サービス — WBS 骨格

「AI を活用した PMO」を顧客企業に導入・支援するコンサルティングサービスの全体 WBS。
**オファリング資料作成 〜 導入完了** までを1本のフローで定義。

> たたき台。Keita のレビュー後、フェーズ単位で詳細化する。

---

## 全体フェーズマップ

```
[事前準備]                [営業活動]                  [プロジェクト実行]
┌─────────┐ ┌──────────────┐ ┌─────────────────────────────┐
│Phase 0  │→│Phase 1, 2, 3 │→│Phase 4, 5, 6, 7              │
│オファ   │ │リード→提案→  │ │KO→アセス→PoC→本番導入→引渡 │
│リング   │ │ 受注          │ │                              │
└─────────┘ └──────────────┘ └─────────────────────────────┘
   3〜4週       3〜6ヶ月            4〜9ヶ月
```

## Phase 0: オファリング準備（社内整備）

| WBS | タスク | 成果物 | 期間目安 |
|-----|-------|------|--------|
| 0.1 | サービスコンセプト確定 | サービス紹介1枚 | 1週 |
| 0.2 | ターゲット顧客セグメント定義 | ペルソナ・規模別ICP | 3日 |
| 0.3 | オファリング資料作成 | 提案ピッチ deck（15〜25p） | 1〜2週 |
| 0.4 | 価格モデル設計（[[pricing]]） | 価格表・契約パターン3種 | 1週 |
| 0.5 | デリバリーキット整備 | ヒアリングシート・診断テンプレ・WBSテンプレ | 1週 |
| 0.6 | 事例コンテンツ準備 | 仮想ケース or POC事例 | 並行 |
| 0.7 | 競合・市場リサーチ（[[competitors-research]]） | 競合マトリクス・ポジショニング | 1〜2週 |

**コンサルタントの役割**: サービス設計者・マーケ。市場・競合リサーチ含む。

## Phase 4: キックオフ・プロジェクト立ち上げ

| WBS | タスク | 成果物 |
|-----|-------|------|
| 4.1 | プロジェクト体制構築（[[team-structure]]） | 体制図・RACI |

**コンサルタントの役割**: プロジェクトマネージャ（最初は人間が PMO 自身として動く）。

## Phase 5: PoC（パイロット導入）

1〜2 プロジェクトに絞って AI PMO を実投入し、効果を測る。

| WBS | タスク | 成果物 |
|-----|-------|------|
| 5.1 | パイロット対象プロジェクト選定 | 選定基準・対象リスト |
| 5.2 | AI エージェント構成カスタマイズ | エージェント設定（[[pmo-roles-map]] 参照） |
| 5.3 | 既存ツール連携（Jira/Backlog/Slack 等） | 連携設定 |
| 5.4 | 運用ルール策定（誰が何をいつ承認） | 運用ガイド |
| 5.5 | 顧客側担当者トレーニング | ハンズオン資料・録画 |
| 5.6 | PoC 実施（4〜8週） | 週次レポート・運用ログ |
| 5.7 | 効果測定 | KPI 計測レポート |
| 5.8 | PoC 振り返り・改善提案 | 振り返り資料・拡張提案 |

**コンサルタントの役割**: AI PMO の伴走者・チューニング担当。

---

## Phase 6: 本番導入・展開

| WBS | タスク | 成果物 |
|-----|-------|------|
| 6.1 | 展開計画策定（対象 PJ 選定・順序） | 展開ロードマップ |
| 6.2 | 標準運用ルール文書化 | 運用マニュアル |
| 6.3 | 既存 PMO 組織との役割再整理 | 役割分担表（人間 vs AI） |
| 6.4 | 段階展開（波状的に複数 PJ へ） | 各 PJ 導入記録 |
| 6.5 | チェンジマネジメント | 説明会・FAQ・苦情対応窓口 |
| 6.6 | ガバナンス設計（KPI 監視・継続改善） | ガバナンス資料 |

**コンサルタントの役割**: チェンジマネージャ・運用設計者。組織抵抗への対処が最大の山場。

## Phase 7: 定着・引き渡し（導入完了）

| WBS | タスク | 成果物 |
|-----|-------|------|
| 7.1 | 運用安定化確認 | 安定化チェックリスト |
| 7.2 | 顧客側運用責任者の育成 | 認定担当者リスト |
| 7.3 | ナレッジ移管 | 運用ナレッジベース |
| 7.4 | 卒業 / 保守契約への移行 | 保守契約 or 引き渡し完了報告 |
| 7.5 | 導入完了報告 | 成果報告書 |

**コンサルタントの役割**: 引き継ぎ役。卒業前提で「自走できる組織」に仕上げる。

## 規模別カスタマイズ方針

| 規模 | Phase 0〜3 | Phase 4〜5 | Phase 6 | Phase 7 |
|------|---------|---------|------|------|
| 小規模 (〜50名) | パッケージ販売型（フェーズ短縮） | PoCなしで即本番 | スコープ限定 | 3〜6ヶ月で卒業 |
| 中規模 (50〜500名) | 標準フロー | PoC 1〜2 PJ | 段階展開 | 6〜12ヶ月で卒業 |
| 大規模 (500名〜) | 提案コンペ前提・長期化 | PoC 3〜5 PJ | 部門単位で展開 | 1〜2年伴走 |

詳細は [[case-studies/small]] / [[case-studies/medium]] / [[case-studies/large]] で。
