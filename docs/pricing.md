# プライシング — どのくらいのフィーでやるべきか

AI PMO 導入支援サービスの価格モデル設計。
**売り方によって価格構造が変わる** ので、契約パターン別に骨子を整理。

> 骨格のみ。Keita レビュー後、相場・案件履歴で具体額に落とす。

---

## 1. 想定する3つの価格モデル

| モデル | 課金構造 | 向く案件 | 売り手のリスク |
|------|-------|------|---------|
| **A. プロジェクト型（一括）** | 期間 × 人月単価 + ライセンス | 中〜大規模の導入案件 | 工数膨張リスクは売り手 |
| **B. サブスク型（月額）** | 月額固定（SaaS + サポート） | 小規模・継続利用 | 解約リスク |
| **C. 成果報酬型（一部）** | KPI 達成度に応じた追加報酬 | 中規模で効果が測れる案件 | 効果の測定難 |

**推奨**: A をベースに、運用フェーズで B に移行。C は実績ある顧客で追加オプション。

---

## 2. プロジェクト型の構成要素

```
案件総額 = 初期費用 + 月次費用 + LLM 利用料
```

### 初期費用（一括）
- アセスメント（[[service-wbs]] Phase 2）: ¥3M〜¥10M
- 提案・設計（Phase 3〜4）: ¥5M〜¥20M
- PoC 構築（Phase 5）: ¥5M〜¥30M
- 本番導入（Phase 6）: ¥10M〜¥80M

### 月次費用（運用）
- 伴走支援（Phase 5〜7）: ¥1M〜¥5M / 月
- ライセンス・保守: ¥500K〜¥3M / 月

### LLM 利用料（パススルー or 上乗せ）
- 顧客契約 or コンサル契約に組込
- 上乗せマージン 10〜20% が業界相場

## 3. 規模別の総額レンジ（仮置き）

| 規模 | 初期費用 | 月次（運用） | 12ヶ月 TCO（目安） |
|----|------|--------|------------|
| 小規模 (〜50名) | ¥3M〜¥8M | ¥300K〜¥800K | ¥7M〜¥18M |
| 中規模 (50〜500名) | ¥15M〜¥40M | ¥1M〜¥3M | ¥30M〜¥80M |
| 大規模 (500名〜) | ¥50M〜¥200M | ¥3M〜¥10M | ¥100M〜¥400M |

**根拠**: 大手コンサルファームの DX 案件相場（人月 ¥1.5M〜¥3M × チーム規模）を参照。
**注**: 数字は仮置き。要市場調査（[[competitors-research]]）。

## 5. プライシング戦略の論点

- **初期は値引きで実績作り** — 最初の3〜5社は通常価格の30〜50%
- **成果事例ができたら標準価格に戻す** — 価値訴求型へ
- **競合（[[competitors-research]]）より高めに置くか** — 「AI 専門特化」のポジショニングで差別化
- **大手 SIer/コンサルとの再販パートナーシップ** — フロントは大手、実装は当方の構造で B2B2C 化

## 6. 顧客への価格説明テンプレ

```
[Phase] アセスメント
└ 期間: 4週 / 人員: 2名（シニア + アナリスト）
└ 費用: ¥XXX 万円

[Phase] 提案・設計
└ 期間: 6週 / 人員: 3名
└ 費用: ¥XXX 万円

[Phase] PoC（パイロット導入）
└ 期間: 8週 / 人員: 4名
└ 費用: ¥XXX 万円

[Phase] 運用伴走
└ 月額: ¥XXX 万円（最低6ヶ月）

LLM 利用料: 別途実費 + マージン15%
```
