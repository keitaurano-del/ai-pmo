# 案件チーム体制

AI PMO 導入支援案件を遂行するためのメンバー体制。フェーズと規模で人数・スキルセットが変わる。

> 骨格のみ。Keita レビュー後、実案件で詳細化。

---

## 1. 必要なロール（5種）

| ロール | 主担当 | スキル要件 |
|-----|------|-------|
| **エンゲージメントリード (EL)** | 顧客窓口・契約・最終責任 | PMO 経験 10年+、コンサル経験 |
| **シニアコンサルタント (SC)** | フェーズ責任・提案リード | PMO + AI 知識、ファシリ |
| **コンサルタント (C)** | 実作業・ドキュメント・分析 | PJ管理経験、AI基礎理解 |
| **AI エンジニア (AE)** | エージェント実装・連携 | LLM/プロンプト、Python/TS |
| **データエンジニア (DE)** | データ連携・ETL・KPI集計 | Jira/Backlog API、BI |

---

## 2. 規模別のチーム編成

### 小規模案件（〜50名顧客 / ¥7〜18M）

| ロール | 人数 | 稼働率 |
|------|---|------|
| EL | 1 | 10% |
| SC | 1 | 50% |
| AE | 1 | 50% |

**合計: 3名 / 1.1人月相当**。EL 兼任で運用、PoC 省略のパッケージ提供。

---

### 中規模案件（50〜500名顧客 / ¥30〜80M）

| ロール | 人数 | 稼働率 |
|------|---|------|
| EL | 1 | 20% |
| SC | 1 | 80% |
| C | 1〜2 | 100% |
| AE | 1 | 80% |
| DE | 1 | 50% |

**合計: 5〜6名 / 3〜4人月**。標準フローを完走。

---

### 大規模案件（500名〜顧客 / ¥100M〜）

| ロール | 人数 | 稼働率 |
|------|---|------|
| EL | 1 | 50% |
| SC | 2 | 100% |
| C | 3〜5 | 100% |
| AE | 2 | 100% |
| DE | 2 | 100% |
| 専任 PM（社内） | 1 | 100% |

**合計: 11〜13名 / 10人月+**。部門横断展開、長期伴走。

---

## 3. フェーズ別の主担当マップ

| フェーズ | リード | サブ |
|------|-----|----|
| Phase 0 オファリング | EL + SC | マーケ |
| Phase 1 営業 | EL | SC |
| Phase 2 アセスメント | SC | C |
| Phase 3 提案・受注 | SC + EL | C, AE |
| Phase 4 キックオフ | SC | C, AE, DE |
| Phase 5 PoC | AE + C | SC, DE |
| Phase 6 本番導入 | SC | C, AE, DE |
| Phase 7 引き渡し | SC | C |

[[service-wbs]] と対応。

---

## 4. スキル獲得・育成戦略

- **EL 候補**: 大手コンサル/SIer の PMO 経験者を採用
- **AE 候補**: AI スタートアップ・LLM 系出身者を採用 or パートナー化
- **C 候補**: 既存 SIer/IT コンサル経験者を内製育成
- **AI リテラシー研修**: 全メンバー必須（プロンプト基礎・ハルシネ対策）

---

## 5. パートナー戦略

自社だけで揃わない場合の補完：

| 領域 | パートナー候補 |
|----|------|
| 大手顧客の窓口 | 大手コンサル/SIer と再販契約 |
| AI 実装力 | AI スタートアップ・フリーランス AE |
| LLM ベンダー連携 | Anthropic / OpenAI / AWS / Azure の認定パートナー化 |
| 業界特化知見 | 金融 / 医療 / 公共 の業界専門ファーム |
