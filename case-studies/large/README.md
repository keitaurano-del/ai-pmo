# 大規模ケース（骨組みのみ）

## 想定設定（暫定、Keita 確定待ち）

- クライアント企業: 従業員 1,000名超 のエンタープライズ（金融・製造・通信 など）
- 既存 PM/PMO 体制: 全社 PMO 組織あり（20〜50名）、複数事業部に PJ 多数
- AI PMO 導入対象: 全社横断のデジタル変革プログラム（10〜20 PJ 同時並走）
- 導入エージェント（推奨）: 全 5エージェント + オーケストレーター
- 導入形態の特殊性: オンプレ / 専用 VPC 構成、強い監査対応・コンプラ要件

## ディレクトリ構造（雛形）

`client-env/` 配下は medium 同じ構造で空ディレクトリのみ作成済。
ただし大規模ケースでは「プログラム全体」「個別 PJ」の2階層となる可能性が高く、構造そのものが medium と異なる。

## 大規模特有の論点（要 Keita レビュー）

- ガバナンス重視（監査ログ、変更管理、職務分離）
- 既存 PMO 組織との役割再定義・人員再配置
- セキュリティ・データ主権（オンプレ or 専用クラウド）
- 多通貨・多拠点 PJ への対応

medium と small が固まった後、大規模特有の組織モデルを整理してから着手する。
