# 中規模ケース — アクロス・システムズ株式会社

中堅 SIer（社員300名・PM/PMO 計15名）にて、AI PMO を 1 プロジェクトに PoC 導入したシナリオの仮想検証環境。

## 設定

- **クライアント企業（弊社の客）**: アクロス・システムズ株式会社（架空）
  - 業種: 受託システム開発（SIer）
  - 従業員数: 320名
  - 既存 PMO: 7名体制、複数 PJ 並走中
- **AI PMO 導入対象 PJ**: NEXUS-RTL（大手小売チェーン M-Mart 向け基幹システム刷新案件）
- **AI PMO 導入フェーズ**: Phase 1（PoC、〜3ヶ月）
- **AI PMO 導入日**: 2026-04-01
- **本検証環境の基準日**: 2026-05-15

## 検証で確認したいこと

1. **WBS の見え方**: AI PMO 導入後、PJ の WBS がどう構造化・更新されるか
2. **議事録の自動化**: 定例会議の議事録が AI スクライブによってどう生成されるか
3. **課題管理の効率化**: 課題が AI トリアージャーによってどう分類・優先度付けされるか
4. **週次報告書の生成**: AI レポーターが集約データからどんな報告書を出すか
5. **既存 PMO との役割分担**: 何が AI に置換され、何が人間に残るか
