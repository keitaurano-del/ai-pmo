# ケーススタディ

仮想企業に対する AI PMO 導入シナリオを、**企業規模別** に検証する。

## なぜ規模別に分けるか

PMO の課題は企業規模で質的に変わる：

| 規模 | 典型的な PMO 課題 | AI PMO の主な価値 |
|------|---------------|---------------|
| **小規模** (〜50名) | そもそも PMO 機能がない／PM 兼任で手が回らない | 1人 PM の作業負荷を AI で肩代わり、低コスト導入 |
| **中規模** (50〜500名) | プロジェクト横断の標準化が不十分、情報が分散 | エージェント群で複数 PJ の可視化・統制を一気に底上げ |
| **大規模** (500名〜) | 既存 PMO 組織が肥大化、形骸化、報告コスト膨大 | 既存 PMO の業務を AI で置換・効率化、人員再配置 |

導入方式・エージェント構成・課金モデル・ROI ストーリーがすべて変わる前提で、それぞれ別案件として設計する。

## ディレクトリ構成

```
case-studies/
├── small/    # 小規模企業（〜50名）
├── medium/   # 中規模企業（50〜500名）
└── large/    # 大規模企業（500名〜）
```

各ディレクトリには、業種を変えた仮想企業を1社以上立てる。

## 仮想企業の設計テンプレ

各ケーススタディは以下のテンプレで記述する：

```markdown
# 企業概要
- 業種:
- 従業員数:
- 主要事業:
- 既存の PM/PMO 体制:

# 抱えている課題

# AI PMO 導入シナリオ
## Phase 1（初期導入、〜3ヶ月）
## Phase 2（拡張、3〜12ヶ月）
## Phase 3（定着、12ヶ月〜）

# 必要なエージェント構成

# 期待効果（KPI）

# 導入コスト・ROI

# リスク・前提条件
```
