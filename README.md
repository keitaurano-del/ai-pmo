# ai-pmo

企業向け「AI PMO 導入支援」コンサルティングサービスの **コード・成果物・検証環境リポジトリ**。

## ⚠️ ナレッジは Obsidian Vault がマスタ

サービス設計・PMO 役割定義・プライシング・競合調査などの **思考・設計ドキュメントは Obsidian 側**で管理。
このリポには `scripts/sync-from-obsidian.sh` で同期したコピーを `docs/` に置く（直接編集禁止）。

📂 Obsidian マスタ: `~/projects/obsidian-vault/30-Projects/ai-pmo/`

## 仮想検証環境について

`case-studies/medium/` が現状のフラッグシップ。

- **シナリオ**: 中堅 SIer「アクロス・システムズ株式会社」が「大手小売 M-Mart 基幹システム刷新案件 NEXUS-RTL」に AI PMO PoC を導入した想定
- **基準日**: 2026-05-15（PoC 開始から第7週、12週中）
- **検証で確認**: AI PMO 導入後の WBS / 議事録 / 課題管理 / 週次報告書の見え方、既存 PMO との役割分担


## ワークフロー

### 新規クライアント環境を 1 コマンドで作る
```bash
npm install                        # 初回のみ
npm run init -- --interactive      # 対話モード
# あるいは
npm run init -- --slug acme --name "ACME株式会社" --size medium --pm "田中 一郎"
```
→ `case-studies/<slug>/` 配下に 18 ファイルのスケルトンが生成される。詳細は [scripts/README.md](./scripts/README.md)

### Obsidian の知見を取り込む
```bash
npm run sync:obsidian
git add docs/
git commit -m "docs: sync from obsidian"
```

### 仮想環境の成果物を増やす
1. `case-studies/medium/client-env/04-meetings/` に新規議事録を追加
2. `05-issues/` の課題が増えれば issues.csv を更新
3. `07-reports/` に翌週の週次報告書を追加
4. 各ファイル末尾の `Generated by:` / `Last updated:` を更新

### AI PMO エージェントを実装する
`agents/<agent>.md` のプロンプト雛形をベースに Claude API or Claude Code subagent として実装。
