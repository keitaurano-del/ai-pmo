# CLAUDE.md — ai-pmo プロジェクト方針

`/root/projects/CLAUDE.md`（全体方針）を継承。ここではこのプロジェクト固有のルールのみ記載。

## プロジェクト概要

企業向け「AI PMO 導入支援」コンサルティングサービス。
- ナレッジマスタ: Obsidian (`~/projects/obsidian-vault/30-Projects/ai-pmo/`)
- 本リポ役割: エージェント実装・仮想検証環境・顧客提案書テンプレ

## ドキュメント編集方針

### `docs/` 配下は直接編集禁止
- Obsidian がマスタ。`scripts/sync-from-obsidian.sh` で同期生成される
- 編集が必要なら Obsidian 側で編集 → sync 実行 → commit

### `case-studies/medium/` の更新ルール
仮想プロジェクトの「進行」を表現するため、以下の規律で更新する：
- 議事録 (`04-meetings/`): 会議日に対応する `YYYY-MM-DD_<種別>.md` で追加（既存ファイル編集ではなく追加）
- WBS / スケジュール / 報告書: 基準日を進めるときは過去版を残さず上書き（履歴は git で追える）
- 課題管理表 (`05-issues/`): `issues.csv` を Source of Truth とし、`issue-tracker.md` は再生成
- 全ファイル末尾の `Generated by` / `Last updated` を必ず更新

### 凜口調と中立体の使い分け
- README / CLAUDE.md / Obsidian docs: 凜口調可（Keita 向け）
- `case-studies/*/client-env/` 配下: **中立的な丁寧体**（仮想クライアントが見る想定の業務文書）
- `agents/` 配下: 中立的・技術ドキュメント体

## エージェント実装の進め方

1. `agents/<agent>.md` のプロンプト雛形を最新化
2. 実装は ↓ いずれか（着手前に Keita 相談）
   - Claude API 直接呼び出し（独立 Node スクリプト）
   - Claude Code subagent 化（`~/.claude/projects-meta/agents/` に追加）
   - 既存の dev-logic / marketing 等 subagent を流用
3. 仮想検証環境 (`case-studies/medium/client-env/`) を入力データとして動作確認

## デプロイ・運用

現状ローカル運用のみ。本番デプロイなし。
将来：顧客環境への配布形態は Phase 2 で決定（SaaS / オンプレ / セルフホスト）。

## push 方針

- ドキュメント追加・編集の commit / push は凜判断で自律実行 OK
- agents/ のプロンプト雛形変更は凜判断 OK
- 実コード（API 呼び出し・統合スクリプト）を追加する場合は事前に Keita 確認
